## **1. Theoretical Foundation**

### **Introduction to The Nothingness Effect**

*The Nothingness Effect* is a theoretical framework that explores reality through dynamic interactions of abstract mathematical constructs. At its core is the **Flowpoint function (fp)**—a function that encapsulates the oscillatory behavior between positive and negative values. This oscillation provides a dynamic equilibrium that can be used to model both the emergence of π and the behavior of trigonometric functions without directly using classical definitions.

### **Flowpoint Function and Trigonometric Modeling**

#### **Flowpoint Function (\( fp \))**

The Flowpoint function embodies a key oscillatory property:
  
\[
fp(f) = (f \neq -f)^{(f = -f)}
\]

- **Oscillation:**  
  Each invocation alternates the sign (i.e. \( fp(f) \) and \( fp(-f) \) cancel out in pairs).
- **Equilibrium:**  
  This alternating behavior is essential for balancing series like the Gregory-Leibniz series for π.

#### **Modeling Trigonometric Functions with \( fp \)**

Rather than use standard trigonometric functions directly, the framework uses the Flowpoint function to reproduce their behavior. For example, given a right-angled triangle with:
- \( x \) as the adjacent side,
- \( y \) as the opposite side, and
- \( h \) as the hypotenuse,

the following representations are used:

- **Flowpoint-based Cosine:**
  
  \[
  fp_{\cos}(x, h) = \frac{fp(x)}{fp(h)}
  \]

- **Flowpoint-based Sine:**
  
  \[
  fp_{\sin}(y, h) = \frac{fp(y)}{fp(h)}
  \]

- **Flowpoint-based Tangent:**
  
  \[
  fp_{\tan}(y, x) = \frac{fp(y)}{fp(x)}
  \]

The class **`FlowpointTrigonometry`** (implemented in `fp_trigonometry.py`) encapsulates these methods. High-level methods (such as `cos(theta)`, `sin(theta)`, and `tan(theta)`) compute the triangle sides via standard math and then transform these using the Flowpoint-based functions.

### **Deriving π from Oscillatory Series**

Rather than directly using the conventional value of π, the framework derives π from an adapted version of the Gregory-Leibniz series:

\[
\pi = 4 \sum_{k=0}^{\infty} \frac{(-1)^k}{2k + 1}
\]

In our approach (implemented in **`pi_model.py`**):

- The alternating term \((-1)^k\) is replaced by a Flowpoint-based version \( fp\_value((-1)^k) \), thereby incorporating the oscillatory behavior.
- With a sufficient number of iterations, the sum converges to an approximation of π.

## **4. Functional Overview**

### **`pi_model.py`**

#### **Function `approximate_pi(iterations)`**

- **Purpose:**  
  Compute an approximation of π via a modified Gregory-Leibniz series by incorporating the Flowpoint function.
  
- **Process:**  
  - For each iteration \( k \):
    - Compute the numerator as \( fp\_value((-1)^k) \) using the FlowpointTrigonometry instance.
    - Compute the denominator as \( 2k + 1 \).
    - Accumulate \( \text{term} = \frac{fp\_value((-1)^k)}{2k+1} \).
  - Multiply the sum by 4 to yield the final approximation.

### **`test_fp_pi_model.py`**

#### **Unit Tests**

- **Test Cases:**  
  - **Low Iterations:** Check that the π approximation is roughly close to π for very few iterations.  
  - **Medium Iterations:** Confirm greater accuracy (e.g., matching up to three decimal places) with around 1,000 iterations.
  - **High Iterations:** Validate that for 100,000 (or more) iterations, the approximation converges to within five decimal places of π.
  
- **Visualization:**  
  The test script also generates static and animated plots (using Matplotlib) displaying the evolution of the approximation and the absolute error.

### **`simulate_pi_model.py`**

#### **Simulation Process**

1. **Data Collection:**
   - Iterate from 1 to a maximum iteration count (e.g., 100,000) in steps.
   - For each iteration count, compute the π approximation and its error relative to `math.pi`.

2. **Plotting:**
   - Create a two-panel plot:
     - One panel shows the approximated π value converging to the actual π value.
     - The other panel shows the decreasing error over iterations.
     
3. **Animation and Data Saving:**
   - Animate the evolution of the approximation.
   - Save static plots (PNG) and animations (MP4 or GIF) to the same directory.
   - Optionally, output a CSV file with the computed data.
