## **1. Theoretical Foundation**

### **Introduction to the Flowpoint Function**

The **Flowpoint (fp)** encapsulates the dynamic fluctuation between positive and negative states, serving as a mediator that maintains equilibrium within dynamic systems. It is defined as:

\[
fp(f) = (f \neq -f)^{(f = -f)}
\]

- **Oscillation Property**:
  \[
  f_{n+1} = -f_n
  \]
  The function alternates between \( f \) and \( -f \), simulating oscillatory behavior.

### **Modeling Trigonometric Functions Using fp**

#### **1. Modeling \( \cos(\theta) \) Using fp**

**Objective**: Demonstrate that the Flowpoint function can model \( \cos(\theta) \).

**Approach**:

- **Definition of \( \cos(\theta) \)** in terms of a right-angled triangle:
  \[
  \cos(\theta) = \frac{x}{h}
  \]
  where \( x \) is the adjacent side and \( h \) is the hypotenuse.

- **Flowpoint Representation**:
  \[
  fp_{\cos}(x, h) = \frac{fp(x)}{fp(h)}
  \]

- **Oscillation Analysis**:

  - **Even Iterations**:
    \[
    fp_{\cos}(x, h) = \frac{x}{h} = \cos(\theta)
    \]
  
  - **Odd Iterations**:
    \[
    fp_{\cos}(x, h) = \frac{-x}{-h} = \frac{x}{h} = \cos(\theta)
    \]
  
  The negative signs cancel out, ensuring that \( fp_{\cos}(x, h) \) consistently equals \( \cos(\theta) \).

#### **2. Modeling \( \sin(\theta) \) Using fp**

**Objective**: Demonstrate that the Flowpoint function can model \( \sin(\theta) \).

**Approach**:

- **Definition of \( \sin(\theta) \)**:
  \[
  \sin(\theta) = \frac{y}{h}
  \]
  where \( y \) is the opposite side.

- **Flowpoint Representation**:
  \[
  fp_{\sin}(y, h) = \frac{fp(y)}{fp(h)}
  \]

- **Oscillation Analysis**:

  - **Even Iterations**:
    \[
    fp_{\sin}(y, h) = \frac{y}{h} = \sin(\theta)
    \]
  
  - **Odd Iterations**:
    \[
    fp_{\sin}(y, h) = \frac{-y}{-h} = \frac{y}{h} = \sin(\theta)
    \]
  
  The negative signs cancel out, ensuring that \( fp_{\sin}(y, h) \) consistently equals \( \sin(\theta) \).

#### **3. Modeling \( \tan(\theta) \) Using fp**

**Objective**: Demonstrate that the Flowpoint function can model \( \tan(\theta) \).

- **Definition**:
  \[
  \tan(\theta) = \frac{y}{x}
  \]
  where \( y \) is the opposite side and \( x \) is the adjacent side.

- **Flowpoint Representation**:
  \[
  fp_{\tan}(x, y) = \frac{fp(y)}{fp(x)} = \tan(\theta)
  \]

- **Oscillation Analysis**:
  Similar to the above, the negative signs cancel out during odd iterations, resulting in a consistent value matching \( \tan(\theta) \).

## **3. Script Structure**

### **1. `fp_trigonometry.py`**

- **Purpose**:  
  Implements the `FlowpointTrigonometry` class, which models \( \cos(\theta) \), \( \sin(\theta) \), and \( \tan(\theta) \) using the Flowpoint function.

- **Components**:

  - **Class**: `FlowpointTrigonometry`
    - **Methods**:
      - `__init__(self, smoothing_window=10)`: Initializes the class with a smoothing window parameter.
      - `fp_value(self, f)`: Generates oscillatory values using the imported `fp` function.
      - `fp_cos(self, x, h)`: Models \( \cos(\theta) \).
      - `fp_sin(self, y, h)`: Models \( \sin(\theta) \).
      - `fp_tan(self, y, x)`: Models \( \tan(\theta) \).
      - `cos(self, theta)`: Calculates cosine using Flowpoint-based methods.
      - `sin(self, theta)`: Calculates sine using Flowpoint-based methods.
      - `tan(self, theta)`: Calculates tangent using Flowpoint-based methods.
      - `cos_sin(self, theta)`: Calculates the sum of cosine and sine using Flowpoint-based methods.
      - `pi(self, precision=1e-4, max_iterations=100000)`: Approximates π using the Flowpoint function.
      - `pi_value(self)`: Returns the approximated value of π.

- **Import Details**:

  - Import the `fp` function and `fp_pi` model from the specified path:
  
    ```python
    import os
    import sys
    import math
    
    parent_dir = os.path.dirname(os.path.dirname(os.path.abspath(__file__)))
    sys.path.insert(0, parent_dir)
    
    from flowpoint.flowpoint import fp
    from flowpoint_pi.pi_model import fp_pi
    ```

## **4. Functional Overview**

### **`fp_trigonometry.py`**

#### **Class `FlowpointTrigonometry`**

1. **`__init__(self, smoothing_window=10)`**

   - **Purpose**:  
     Initializes the `FlowpointTrigonometry` class with a specified smoothing window to control the responsiveness of oscillatory values.

   - **Parameters**:
     - `smoothing_window` (int): Number of iterations to average oscillatory values for smoothing.

   - **Logic**:  
     Sets up any necessary initial parameters and data structures.

2. **`fp_value(self, f)`**

   - **Purpose**:  
     Obtains oscillatory values for input `f` using the imported `fp` function.

   - **Parameters**:
     - `f` (float): Input value for the Flowpoint function.

   - **Returns**:
     - `float`: The next oscillatory value in the sequence for input `f`.

   - **Logic**:
     - Initializes a generator for each unique input `f`.
     - Retrieves and returns the next value from the generator.

3. **`fp_cos(self, x, h)`**

   - **Purpose**:  
     Models \( \cos(\theta) \) using the Flowpoint function.

   - **Parameters**:
     - `x` (float): Adjacent side length of a right-angled triangle.
     - `h` (float): Hypotenuse length of a right-angled triangle.

   - **Returns**:
     - `float`: The Flowpoint-based value of \( \cos(\theta) \).

   - **Logic**:
     - Computes `fp_value(x)` and `fp_value(h)`.
     - Calculates \( fp_{\cos}(x, h) = \frac{fp(x)}{fp(h)} \).

4. **`fp_sin(self, y, h)`**

   - **Purpose**:  
     Models \( \sin(\theta) \) using the Flowpoint function.

   - **Parameters**:
     - `y` (float): Opposite side length of a right-angled triangle.
     - `h` (float): Hypotenuse length of a right-angled triangle.

   - **Returns**:
     - `float`: The Flowpoint-based value of \( \sin(\theta) \).

   - **Logic**:
     - Computes `fp_value(y)` and `fp_value(h)`.
     - Calculates \( fp_{\sin}(y, h) = \frac{fp(y)}{fp(h)} \).

5. **`fp_tan(self, y, x)`**

   - **Purpose**:  
     Models \( \tan(\theta) \) using the Flowpoint function.

   - **Parameters**:
     - `y` (float): Opposite side length of a right-angled triangle.
     - `x` (float): Adjacent side length of a right-angled triangle.

   - **Returns**:
     - `float`: The Flowpoint-based value of \( \tan(\theta) \).

   - **Logic**:
     - Computes `fp_value(y)` and `fp_value(x)`.
     - Calculates \( fp_{\tan}(y, x) = \frac{fp(y)}{fp(x)} \).

6. **`cos(self, theta)`**

   - **Purpose**:  
     Calculates cosine using Flowpoint-based methods.

   - **Parameters**:
     - `theta` (float): Angle in radians.

   - **Returns**:
     - `float`: Approximated cosine value.

   - **Logic**:
     - Sets hypotenuse `h = 1.0`.
     - Calculates adjacent side `x = math.cos(theta) * h`.
     - Calls `fp_cos(x, h)` to obtain the Flowpoint-based cosine value.

7. **`sin(self, theta)`**

   - **Purpose**:  
     Calculates sine using Flowpoint-based methods.

   - **Parameters**:
     - `theta` (float): Angle in radians.

   - **Returns**:
     - `float`: Approximated sine value.

   - **Logic**:
     - Sets hypotenuse `h = 1.0`.
     - Calculates opposite side `y = math.sin(theta) * h`.
     - Calls `fp_sin(y, h)` to obtain the Flowpoint-based sine value.

8. **`tan(self, theta)`**

   - **Purpose**:  
     Calculates tangent using Flowpoint-based methods.

   - **Parameters**:
     - `theta` (float): Angle in radians.

   - **Returns**:
     - `float`: Approximated tangent value.

   - **Logic**:
     - Calculates `x = math.cos(theta)` and `y = math.sin(theta)`.
     - Calls `fp_tan(y, x)` to obtain the Flowpoint-based tangent value.

9. **`cos_sin(self, theta)`**

   - **Purpose**:  
     Calculates the sum of cosine and sine using Flowpoint-based methods.

   - **Parameters**:
     - `theta` (float): Angle in radians.

   - **Returns**:
     - `float`: Sum of cosine and sine values, constrained to \([-2, 2]\).

   - **Logic**:
     - Computes `cos(theta)` and `sin(theta)` using Flowpoint-based methods.
     - Sums the results and clamps the value within \([-2, 2]\).

10. **`pi(self, precision=1e-4, max_iterations=100000)`**

    - **Purpose**:  
      Approximates π using the Flowpoint function.

    - **Parameters**:
      - `precision` (float): Desired precision for the approximation.
      - `max_iterations` (int): Maximum number of iterations.

    - **Returns**:
      - `float`: An approximation of π.

    - **Logic**:
      - Calls the imported `fp_pi(max_iterations)` function to compute π.

11. **`pi_value(self)`**

    - **Purpose**:  
      Returns the approximated value of π.

    - **Returns**:
      - `float`: The approximated value of π.

    - **Logic**:
      - Calls the `pi()` method with default parameters.
