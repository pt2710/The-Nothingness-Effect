# PGQENN runtime

Canonical PGQENN composes QENN/DTQC with the pinned MPL-TC dependency. MPL-TC
supplies the verified positive prime/motif data and four finite TC streams:
pure even lifts, first-order odd primes, least-prime-factor odd composites, and
mixed even composites.

The negative number spectrum is a separate TNE neural bridge. Each positive
value `n` receives a Flowpoint partner `-n`; the partner map squares to the
identity. Negative 3D coordinates and local weights may grow asymmetrically and
are never described as upstream MPL-TC semantics.

Both `test/` and `simulation/` producers generate node/edge tables, static 3D
views, adjacency comparisons, simultaneous signed-growth GIFs and signed
message-propagation GIFs. The model also exposes positive-only and prime-only
source-removal modes. These finite artifacts are computational evidence, not
formal proof or an empirical performance claim.
